## 一、背景与目标

### 1.1 现状

当前历史记录面板仅记录**正文内容的不同创作版本**（编辑、AI润色、手动保存等），用户无法回溯创作过程中的关键阶段数据（设定、角色、大纲等）。一旦进入下一阶段，前一阶段的数据无法查看或恢复。

### 1.2 目标

- 在历史记录面板中自动记录创作过程中各阶段的完整快照
- 支持用户预览任意阶段快照的结构化内容
- 支持两种还原模式：回退到此阶段 / 仅更新资料
- 过程稿与正文版本在同一时间线混排，按时间倒序排列

### 1.3 非目标

- 过程稿版本对比（diff）— 本期不做
- 过程稿独立筛选/过滤 — 本期不做
- 手动添加过程稿标签或备注 — 本期不做
- Agent 流程中「已有数据时询问用户」的交互 — 后续迭代，不在本功能范围

## 三、数据模型

### 3.1 HistoryItem 接口

```typescript
interface HistoryItem {
  id: string;                           // 唯一标识，格式 h${timestamp}
  timestamp: Date;                      // 记录时间
  type?: "content" | "stage";          // 区分正文版本 / 过程稿；缺省为 "content"

  // ─── 正文版本字段 ───
  chapterId: string;                    // 所属章节 ID
  chapterTitle: string;                 // 章节标题
  content: string;                      // 正文 HTML 内容
  wordCount: number;                    // 字数
  action: "edit" | "ai_rewrite" | "ai_polish" | "ai_condense"
        | "ai_atmosphere" | "manual_save"   // 正文操作类型
        | "stage_settings" | "stage_characters" | "stage_outline"
        | "stage_brief" | "stage_script";   // 过程稿操作类型

  // ─── 过程稿独有字段（type === "stage" 时有效） ───
  stageKey?: string;                    // 对应 agentStageData 的 key
  stageSummary?: string;                // 时间线摘要文字
  stageSnapshot?: Record<string, any>;  // 完整数据快照，格式 { [key]: data }
}
```

### 3.2 快照存储格式

`stageSnapshot` 的结构为 `{ [stageKey]: data }`，即以 stageKey 为顶层属性：

| stageKey | snapshot 结构 |
|----------|--------------|
| `settings` | `{ settings: { "创作设定": [{label, value}], "写作要素": [{label, value}], ... } }` |
| `characters` | `{ characters: { femaleLead: {name, identity, personality, ...}, maleLead: {...}, supporting: [...], relationships: string } }` |
| `outline` | `{ outline: { structure: string, totalChapters: number, estimatedWords: string, chapters: [{title, summary, keyEvent, season}] } }` |
| `videoBrief` | `{ videoBrief: { confirmedParams: [{label, value}], sellingPoints: {core, secondary, differentiator}, audience: {...}, directions: [...] } }` |
| `videoScript` | `{ videoScript: { characters: [{name, description}], scenePlans: [...], scenes: [{act, heading, characters, visual, dialogue, ...}] } }` |

### 3.3 存储规则

- 总条数上限：**50 条**（过程稿 + 正文版本合计），超出后淘汰最早的
- 自动生成：`setAgentStageData` 被调用且 key 在 `stageLabels` 映射中时，自动创建过程稿条目
- 前端存储：数据保存在 Zustand store 的 `historyItems` 数组中，页面刷新后丢失（当前无持久化）

### 4.5 还原交互

#### 4.5.1 设计原则

还原操作的语义是单一的：**把某个阶段的数据恢复到快照状态。** 至于恢复之后怎么继续创作，属于 Agent 流程本身的职责，不在还原环节中处理。

核心原则：
1. **不清除已有数据** — 还原操作只恢复快照数据、回退阶段指针，不删除任何下游内容
2. **自然覆盖** — 重新生成时新数据自然覆盖旧数据，旧数据在覆盖前仍可见
3. **对话流不变** — 还原操作不修改、不删除、不标记对话流中的卡片

#### 4.5.2 正文版本还原

点击「还原此版本」→ 立即还原，关闭面板，Toast 提示「已恢复到历史版本」。

- 按钮样式：`bg-gray-700 text-white`
- 最新一条不可还原（`disabled`，`bg-gray-200 text-gray-400 cursor-not-allowed`）

#### 4.5.3 过程稿还原

点击「还原此版本」→ 弹出模式选择 Modal，不直接还原。

- 按钮样式：`bg-indigo-600 text-white`（与正文版本颜色区分）

**Modal 设计：**

```
┌────────────────────────────────────────────────┐
│  还原过程稿                                     │
│  选择还原方式，恢复「确认创作设定」快照            │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │ 回退到此阶段                               │  │
│  │ 恢复此阶段数据，回到该创作阶段重新开始。     │  │
│  │ 后续已生成的内容暂时保留，重新生成时自然覆盖  │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │ 仅更新资料                                 │  │
│  │ 恢复此阶段数据，不改变当前创作阶段，         │  │
│  │ 不影响后续已生成的内容                       │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│                   取消                          │
└────────────────────────────────────────────────┘
```

- Modal 尺寸：`w-[400px]`，居中浮层，遮罩 `bg-black/20`
- 两个选项为卡片式按钮，hover 时 `border-indigo-300 bg-indigo-50/30`

#### 4.5.4 两种还原模式的业务逻辑

**模式 A：回退到此阶段（restart）**

| 步骤 | 操作 |
|------|------|
| 1 | 从 `stageSnapshot` 恢复 `agentStageData[stageKey]` |
| 2 | 回退 `creationStage` 到该阶段对应的阶段号 |
| 3 | **不清除**任何下游数据（角色/大纲/正文均保留） |

阶段号映射：

| stageKey | 回退到的 creationStage |
|----------|----------------------|
| settings | 1 |
| characters | 3 |
| outline | 4 |
| videoBrief | 2 |
| videoScript | 3 |

**数据保留与覆盖机制：**

还原后，下游阶段的数据暂时保留（编辑器、查看资料仍显示旧数据）。当 Agent 流重新走到该阶段并生成新数据时，`setAgentStageData` 自然覆盖旧数据。这意味着：

- 用户回退到设定阶段 → 角色/大纲/正文**仍可见**，不会被清空
- 用户可以选择保留现有角色，只修改大纲 → 在 Agent 流中决定
- 用户选择重新生成角色 → 新角色数据覆盖旧角色数据

> **注意**：当前 Agent 流是线性的，不支持「遇到已有数据时询问用户」。此能力为后续迭代内容（见第九章后续规划 P1）。当前回退到某阶段后，Agent 会按原有流程从该阶段继续，生成的新数据自然覆盖。

**模式 B：仅更新资料（update）**

| 步骤 | 操作 |
|------|------|
| 1 | 从 `stageSnapshot` 恢复 `agentStageData[stageKey]` |
| 2 | 不改变 `creationStage` |
| 3 | 不影响任何下游数据和正文 |

两种模式的**唯一区别**是是否回退 `creationStage`。数据操作完全一致：只恢复快照数据，不清除、不覆盖任何其他数据。

#### 4.5.5 还原后效果

| 区域 | 行为 |
|------|------|
| 编辑器 | 根据 `agentStageData` 和 `creationStage` 自动更新 |
| 查看资料面板 | 自动同步（读取同一 store 数据） |
| 对话流 | **不修改、不删除、不标记失效**，保持原样 |
| 下游数据 | 暂时保留，Agent 重新生成时自然覆盖 |

#### 4.5.6 还原后 Toast 提示

| 模式 | Toast 文案 |
|------|-----------|
| restart | 已回退到此阶段，可重新创作 |
| update | 已更新创作资料 |

## 六、边界情况与约束

| 场景 | 处理方式 |
|------|---------|
| 重复生成同一阶段 | 每次都创建新的过程稿条目，旧的保留可回溯 |
| 总条数超限（>50） | 淘汰最早的条目（FIFO） |
| stageSnapshot 为空 | 降级展示 fallback（图标 + 摘要 + 时间戳），还原时仅回退阶段指针 |
| 还原到某个设定后再次修改 | 新的修改会自动创建新的过程稿条目 |
| 回退阶段后下游数据 | 保留不删除，Agent 重新生成时自然覆盖 |
| 新建文档 | 初始无历史记录，面板显示「暂无历史记录，创作过程将自动保存」 |
| 页面刷新 | 当前无持久化，历史记录丢失（待后续接入持久化存储） |
| 未生成的阶段 | 不在时间线中出现（只有 `setAgentStageData` 被调用才记录） |

## 七、前端接口说明

### 7.1 Store 方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `setAgentStageData` | `(key: string, data: any) => void` | 更新阶段数据；若 key 在 stageLabels 中，自动创建过程稿历史条目 |
| `addHistoryItem` | `(item: Omit<HistoryItem, "id" \| "timestamp">) => void` | 手动添加历史条目（含自动生成 id 和 timestamp） |
| `restoreFromHistory` | `(historyId: string) => void` | 还原正文版本（直接覆盖 chapter content + wordCount） |
| `restoreStageFromHistory` | `(historyId: string, mode: "restart" \| "update") => void` | 还原过程稿。两种模式区别仅在于是否回退 creationStage，均不清除下游数据 |

### 7.2 自动记录的阶段 key

在 `setAgentStageData` 内部，以下 key 会自动触发过程稿记录：

```typescript
const stageLabels = {
  settings: "创作设定",
  characters: "角色设定",
  outline: "大纲",
  videoBrief: "创意Brief",
  videoScript: "分幕剧本",
};
```

### 7.3 组件通信

```
TopBar (HistoryPanel)
  ├─ onRestore(id)           → restoreFromHistory → 还原正文
  └─ onRestoreStage(id, mode) → restoreStageFromHistory → 还原过程稿
```
